S3GPIO is Scratch 3 GPIO. That is created by Simon Walters, a Raspberry Pi community member who campaigned for children to learn coding and electronics with Scratch. Scratch was the first foray into the GPIO by Walters in Scratch. Scratch was the first foray into Scratch, based on Scratch 1.4. The old version still exists, but can be used with older versions. In this guide, we will install S3GPIO and learn how to use a high-end electromagnetic sensor as a controller for a DC motor. This is based on our distance with the sensor to a faster speed. We also learn how to create an emergency stop for a motor and incorporate motor control into a code sequence. All of the steps in this tutorial can be used in multiple motors, and we have to fix two more GPIO pins to the L9110S to run the vehicle and control it independently.

Connecting the components are necessary.#

Although the electronic components are relatively simple to connect, the sheer amount of wires might seem confusing to me. Okay, break it down into sections. S3GPIO uses physical pin numbering, not the BCM numbering standard set by the Raspberry Pi Foundation. That means that we have two columns of twenty pins. The two columns start with pin one at the top of the left and pin 2. The left column is the odd column with pins resulting from an odd number pattern: 1,3,5,7,9 etc. The left column is an even column, with numbers progressing 2,4,6,8 etc.

Installing S3GPIO#

  1. Power up your Raspberry Pi and open the terminal. Enter the following command to download the program. Note that the command will download the installer, which generally is very risky for production machines, but it’s safe. wget https://git.io/vMS6T -O isgh8.sh.
  2. Install S3GPIO using the command. sudo bash isgh8.sh rhine sah.
  3. Restart your Raspberry Pi for the change to happen.
  4. Double left click on the S3GPIO icon, and press “End in the terminal” to launch three applications. All the web servers, ScratchGPIO and Chromium and an extension to the existing s3GPIO feature will be launched and it may take a few moments to see S3GPIO.
  5. Click on the map to view inside to enter the online report editor.

Adding more to the code: Create an equator.#

By using the sensor, we can do an intelligent shunt. This sequence will spin the motor in one direction, then reverse before turning the motors over again. The sequence will iterate (loop) 10 times before it ends and it awaits the user to start the sequence again.

  1. From Events, drag What space key pressed and a new sequence of code became available. Change the space key from an r in the drop-down.
  2. From Control, drag repeat 10 and place this under the previous block.
  3. From My Blocks, drag two command blocks and place them in the loop. Turn the second down on the 8-on and the 10-off-on-one.
  4. From Control, drag and place Wait a second and take it to the nearest point in the loop.
  5. Left click on the top of the S3GPIO block and then enter to duplicate. It’ll replace the three blocks we’ve just created. Under the previous, place those new blocks in a loop. In these blocks change the number 8 and the number 10 to the number 11.
  6. Repeat the double-block, and place the newly duplicated three under them, still inside the loop. Set the S3GPIO command so that both of them are off. This project has already appeared in the Linux Magazine issue.
